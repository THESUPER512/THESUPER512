C#第二天

类和名称空间

类(class)

类库引用

类库引用是使用名称空间的物理基础（不同技术类型的项目会引用不同的类库

DLL引用

黑盒引用 无源代码 （缺点：无法进行代码错误修改）由于Dll引用中的类库错误，会导致程序项目的错误依赖

引用System

直接引用System 命名空间

Dll引用必须要有说明文档，没有文档就是抓瞎，里面的类也不知道咋调用

可以进行添加，若类型库中没用想要引用的dll，可以自行上传

NuGet使用（解决复杂的依赖关系）

在NuGet里寻找类库，相对来说比较方便

项目引用

白盒引用 有源代码 （可进行错误代码的修改，主要是因为自己去编写代码

可以引用自己编写好的项目

依赖关系

类（或对象）之间的耦合关系 （高内聚，低耦合）

C#第三天

类、对象、类成员

类是对现实世界事物进行抽象所得到的结果（事物包括实体与逻辑）

建模是一个去伪存真、由表（接口）及里（内部封装逻辑）的过程

类的三大成员

属性 (Property)

存储数据，组合起来表示类或对象当前的状态

方法（Method）

由c语言中的函数（function）进化而来，表示类或对象“能做什么”

工作中90%的时间是和方法打交道，因为他是真正做事、构成逻辑的成员

事件（Event)

类或对象通知其他类或对象的机制，为C#所特有的（java通过其他办法实现这个机制）

善用事件机制很重要

某些特殊类或对象在成员方面的侧重点不同

模型类或对象重在属性（Entity Framework）

工具类或对象重在方法（Math、Console）

通知类或对象重在事件 （各种Timer）

使用new操作符创建类的实例

之后可以进行引用变量操作

静态成员

静态（Static）成员在语义上表示它是“类的成员”

实例成员

实例（非静态）成员在语义上表示他是“对象的成员”

绑定（Binding）指的是编译器如果把一个成员与类或对象关联起来

C#第五天

详解数据类型、变量和对象

类型（type)

数据类型（DataType）

  • 是数据在内存中存储时的”型号”
  • 小内存容纳大尺寸数据时会丢失精确度、发生错误
  • 大内存容纳小尺寸数据会导致浪费
  • 编程语言的数据类型与数据的数据类型不完全相同

强类型语言与弱类型语言的比较

  • C语言示例：if条件
  • javaScript示例：动态类型
  • C#语言对弱类型/动态类型的模仿

C#的强类型体现在数据类型的不可改变的强制性，保证了数据的完整性（如int就是int，long就是long，声明完为int型，就不能改为long型）

javaScript的数据不受数据类型的约束（灵活性较强，弱类型容易导致数据混乱

类型在C#语言中的作用

一个C#类型中包含的信息有：

  • 存储此类型变量所需的内存空间大小
  • 此类型的值可表示的最大、最小值的范围
  • 此类型所包含的成员（如方法、属性、事件等）
  • 此类型有何基类派生而来
  • 程序运行的时候，此类型的变量在分配内存的什么位置
    • Stack简介
    • Stack overflow
    • Heap简介
    • 使用Performance Monitor查看进程的堆内存使用量、
    • 关于内存泄漏
  • 此类型所允许的操作（运算）

一般在软件中经常使用Ulong来表示用户的UID

变量、对象与内存

变量 = 以变量名为所对应的内存地址为起点、以其数据类型所要求的存储空间为长度的一块内存区域

  • 变量定义：

    • 表面上，变量的用途是存储数据
    • 实际上，变量表示了存储位置、并且每个变量都有一个类型，以决定什么样的值能够存入变量
    • 变量一共有七种：
      • 静态变量、实例变量（成员变量，字段），数组元素，值参数，引用参数，输出形参，局部变量。
    • 狭义的变量指局部变量（成员变量，字段），因为其他种类的变量都又自己的约定名称
      • 局部变量就是方法体（函数体）里的声明的变量
    • 变量的声明
      • 有效的修饰符组合opt类型 变量名 初始化器opt

  • 值类型得变量

    • byte/sbyte/short/ushort
    • 值类型没有实例，所谓的”实例”与变量合二为一

  • 引用类型的变量与实例

    • 引用类型变量与实例的关系：引用类型变量里储存的数据是对象的内存地址

  • 局部变量是在stack上分配内存

  • 变量的默认值

  • 常量(值不可改变的变量)

  • 装箱与拆箱(boxing&Unboxing)

变量前加ref修饰 引用参数变量

out double a 输出参数变量

byte是 0255 十进制 0000 0000 1111 1111

const 修饰符 表示后面修饰的数据不可改变

方法

  • 方法(method)的前身是C/C++语言的函数(function)
    • 方法是面向对象范畴的概念，在非面向对象语言中仍然称为函数
    • 使用C/C++语言做出对比

方法

标准函数体

C++是对C的程序时向下兼容的，可以运行，C#和java是在C++的基础上去除了非面向对象的部分。

函数在成为类的成员变量的时候就已经可以被称为方法了

  • 永远都是类(或结构体)最基本的成员之一

    • C#语言中函数不可能独立于类(或结构体)之外
    • 只有作为类(结构体)的成员时才会被称为方法
    • C++中是可以的，成为”全局函数”

  • 是类(或结构体)最基本的成员之一

    • 最基本的成员只有两个——字段与方法(成员变量与成员函数)，本质还是数据+算法
    • 方法表示类(或结构体)“能做什么事情”

  • 为什么需要方法函数

    • 隐藏复杂逻辑
    • 把大算法分解为小算法
    • 复用(reuse,重用)

复用的演示，不用一次更改数据，其他方法可以在别的方法中被使用。

方法的定义与调用

  • 声明方法的语法详解

    • 参见C#语言文档（声明/定义不分家）
    • Parameter全称“formal parameter”形式上的参数，简称“形参”
    • Parameter是一种变量

  • 为方的命名规范

    • 大小写规范
    • 需要以动词或者动词短语作为名

  • 静态(static) 方法和实例方法

  • 调用方法

    • Argument中文C#文档的官方译法“实际参数”，简称“实参”。可以理解为调用方法时的真实条件
    • 调用方法时的argument列表要与定义方法时的parameter列表相匹配
      • C#是强类型语言，argument是值、parameter是变量，值与变量一定要匹配，不然编译器会报错。

错误， double x 是声明变量 ，直接调用x的实际参数就可
